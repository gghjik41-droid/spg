# KODA.md — Проект ПСО Опросники

## Обзор проекта

**Назначение:** PWA-приложение (Progressive Web App) для поисково-спасательных операций. Используется для заполнения опросников при подготовке групп к выходу на задачи, а также как справочник-памятка для старших групп.

**Тип:** Веб-приложение (HTML/CSS/JS), работает как в браузере, так и может быть установлено как PWA.

**Основные возможности:**
- Несколько вкладок-опросников (НА ПСР, МОЛИТВА, ПАМЯТКИ, 222222 и др.)
- Вывод заполненных данных в блок результата с возможностью копирования
- Сохранение данных в localStorage (автосохранение)
- Экспорт/импорт данных в JSON
- Адаптивный дизайн (desktop + mobile)

## Архитектура

### Главная страница (index.html)

Содержит:
- Фиксированную навигационную шапку (`.header-nav`)
- Фрейм для загрузки вкладок (`#survey-frame`)
- Блок результата (`#output`) с кнопкой копирования
- Экран загрузки (loader) при старте
- Индикатор онлайн/офлайн статуса
- Кнопки экспорта/импорта JSON

**Двухколоночный макет:**
- `form-side` — левая колонка с формой
- `result-side` — правая колонка с результатом

**Коммуникация с вкладками:** Через `postMessage`:
- `survey_update` — данные из вкладки в родителя
- `set_height` — высота контента для автоподстройки фрейма
- `hide_result` / `show_result` — скрытие блока результата
- `sync_globals` — синхронизация глобальных данных
- `sync_field` — синхронизация отдельного поля
- `sync_scroll` — синхронизация скролла

### Вкладки (npcr.html, molitva.html и др.)

Каждая вкладка — отдельный HTML-файл со своей логикой. Общие принципы:
- Подключают `common.css`
- Отправляют данные в родительское окно через `postMessage`
- Сохраняют состояние в localStorage
- Управляют высотой через `sendHeight()`

### Общие стили (common.css)

Содержит все CSS, используемые во вкладках:
- Базовые стили (body, h2, label)
- Секции (`.section-block`, `.section-header`, `.section-content`)
- Поля ввода (input, textarea)
- Участники группы (`.member-fields`, `.senior-fields`)
- Специфичные стили для 222222

### НА ПСР (npcr.html)

Основная анкета — карточка группы перед выходом на ПСР.

**Секции:**
- **ДАННЫЕ ШТАБА** — координаты, номер, частота связи, ориентир и азимут
- **ВВОДНЫЕ ПО ПОИСКУ** — ФИО пропавшего, домашнее имя, особенности
- **ОРИЕНТИРОВКА** — загрузка картинок ориентировок (до 10 штук)
- **ЗАДАЧИ** — несколько задач, каждая содержит:
  - Позывной группы
  - Краткая формулировка задачи
  - Состав группы (старший + участники с номерами телефонов)

**Особенности:**
- Множественные задачи (кнопка "Добавить задачу")
- Состав группы: старший группы (№1) + участники (с №2)
- Auto-grow для textarea
- Сохранение состояния секций (открыто/закрыто)
- Загрузка картинок ориентировок с зумом (два пальца)
- Экспорт участников в контакты (vCard)
- IndexedDB для хранения картинок

## Разработка

### Добавление новой вкладки

1. Создать HTML-файл (например, `newtab.html`)
2. Подключить `common.css`
3. Добавить в навигацию `index.html`:
   - Кнопку в `.header-nav`
   - В массив `ORDER`
4. Реализовать базовые функции:
   - `toggleSection()` — переключение секций
   - `sendHeight()` — отправка высоты
   - `syncEverything()` (если нужна синхронизация данных)
   - Сохранение в localStorage

### Скрытие блока результата

Если вкладка не должна показывать результат, добавить функцию:

```javascript
function hideResult() {
    window.parent.postMessage({ type: 'hide_result', hide: true }, '*');
}

window.onload = function() {
    hideResult();
    sendHeight();
};
```

### Синхронизация данных

Для синхронизации данных между вкладками используются следующие сообщения:

- `sync_globals` — отправка глобальных данных (позывной)
- `sync_field` — синхронизация отдельного поля
- `sync_scroll` — синхронизация скролла с блоком результата

Пример получения данных:
```javascript
window.addEventListener('message', function(event) {
    if (event.data.type === 'set_global_data' && event.data.fields) {
        // fields содержит синхронизируемые поля
    }
});
```

### Стили

Все общие стили в `common.css`. При добавлении новых:
- Использовать существующие классы где возможно
- Избегать дублирования
- Для вкладок-справочников добавлять специфичные стили в `<style>` внутри HTML

### Требования к коду

- **Язык:** JavaScript (ES6+), без фреймворков
- **CSS:** ванильный, переменные где уместно
- **HTML:** семантический, доступный
- **Хранение:** localStorage (ключи с префиксами `survey_`, `pso_`), IndexedDB для картинок

## Запуск и сборка

Проект не требует сборки. Открыть `index.html` в браузере.

Для работы PWA (установка на телефон):
- Нужен HTTPS или localhost
- Service Worker регистрируется в `index.html`

## Известные особенности

- Вкладка 222222 существует, но закомментирована в навигации
- Вкладки 333333, 444444, 555555 удалены
- Заголовки `h2` в некоторых вкладках закомментированы (дублируют название вкладки)
- Блок результата показывается по умолчанию, скрывается только если вкладка явно отправляет `hide_result`
- Стрелка раскрытия секций (`.section-toggle`) — CSS-фигура, а не текстовый символ
- Папка `топо` содержит 95 топографических условных знаков (png/jpg)
- Экран загрузки (loader) показывается при старте приложения
- Индикатор онлайн/офлайн отображается в правой части навигации
- Если обнаружено неизвестное изменение в коде, ничего не менять и не исправлять, сначала спросить - нужно ли. Пользователь регулярно вносит изменения вручную по своему разумению
